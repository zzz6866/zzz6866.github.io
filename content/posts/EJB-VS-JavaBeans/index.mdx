> ## EJB (Enterprise JavaBeans)

- Java EE 응용 프로그램 서버의 구성 요소로 여러 가지 유형으로 제공된다.
- 세션 관리, 보안, 트랜잭션(리소스 pooling) 등의 서비스를 제공하는 컨테이너에서 캡슐화된 비즈니스 로직을 제공하고 로직이 교차문제(cross-cutting concerns)가 발생하지 않도록 방지한다.
- 버전이 올라가면서 사용하기가 수월해짐.
- 서버에만 존재하며 Tomcat에는 없다.
- J2EE 아키텍처의 EJB 컨테이너에서 실행.
- 3가지 유형
   1. **Session** : 일부 비즈니스 로직을 제공
   2. **Entity** : 데이터 저장소와 인터페이스를 제공
   3. **Message**-Driven : JMS 메시지를 수신
- J2EE 아키텍처의 EJB 컨테이너에서 실행.

> ## JavaBeans

- 데이터베이스의 데이터를 조회하고 저장하는데 사용되며, MVC 모델의 JSP와 Servlet에 데이터를 전달하여 사용한다.
- setter, getter가 정의된 클래스에서 데이터를 가져올때 사용한다.
- AWT 및 Swing 구성 요소는 Java Bean을 기반
- 직렬화 가능.

> # EJB vs JavaBeans 차이점

| | EJB | JavaBeans |
| ------ | ---------------------------------------- | ---: |
| | 원격 객체로 작동 | 로컬 객체로 작동 |
| | 미들웨어 계층 | 클라이언트/서버 |
| | 세션 빈, 엔티티 빈 | 유형X |
| | 트랜잭션 O | 트랜잭션 X |
| | 개발이 어렵다 | 개발이 쉽다 |
| | JavaBean 사양을 구축하거나 확장하지 않음 | EJB 사양을 사용하여 개발하는 것이 쉽다 |
